# Reels — Jenga Tension Format (Brendan Kane's Jenga Theory)

Brendan Kane framework — inspired từ trò chơi Jenga: 1 MAIN question ở hook giữ toàn bộ video, và mỗi layer câu hỏi phụ (micro-Q) được giải xong lại "rút thanh gỗ" tạo câu hỏi mới. Dùng bởi Veritasium shorts, Mark Rober teaser-first style.

**Core insight**: Viewer không rơi vì tension KHÔNG BAO GIỜ reset về 0 — mỗi answer vừa đóng 1 loop vừa mở loop mới. Khi payoff cuối tie hết các layer, viewer cảm thấy "worth it".

## Structure (6 sections, ~30-60s total)

1. **Hook + MAIN Q** (0-3s) — state câu hỏi chính với stakes
2. **Micro-Q 1** (3-15s) — layer 1 → answer ngắn → introduce Micro-Q 2
3. **Micro-Q 2** (15-30s) — layer 2 → answer ngắn → introduce Micro-Q 3
4. **Micro-Q 3** (30-45s) — layer 3 escalating tension
5. **Final payoff** (45-55s) — resolve MAIN Q, tie 3 layer lại
6. **Soft CTA** (55-60s) — non-autocratic

## Input

1. **Topic** (vd: "AI agent", "Claude Code", "Fine-tune vs Prompting")
2. **Main hook question** (vd: "AI agent có thay được junior dev không?")
3. **Duration** (default 60s)

## Process (Steps)

### Step 1 — Xác định MAIN question

Câu hỏi phải:
- Có stakes thực (viewer care về câu trả lời)
- Không trả lời được trong 5s (cần layer)
- Framing binary hoặc unexpected (vd: "X có thay được Y không?" / "Tại sao X đắt hơn Y?" / "X hay Y?")

Viết 3 phiên bản MAIN Q cho user chọn.

### Step 2 — Plan 3 layer micro-Q

Mỗi layer trả lời 1 sub-aspect của MAIN Q. Thứ tự theo escalation:
- **Layer 1** — aspect dễ nhận ra (surface)
- **Layer 2** — aspect reveal nuance (mid)
- **Layer 3** — aspect kill counter-argument (deep)

Mỗi layer Q phải "grow" từ answer của layer trước — đừng để các layer rời rạc.

### Step 3 — Viết hook + MAIN Q (0-3s)

Hook pattern:
- "[MAIN Q]? Mình test 3 tuần..."
- "Mọi người hỏi: [MAIN Q]. Đây là câu trả lời thật."
- "[MAIN Q]? Câu trả lời không như bạn nghĩ."

Mở bằng câu hỏi, KHÔNG mở bằng answer.

### Step 4 — Micro-Q 1 block (3-15s)

Cấu trúc:
- `[MICRO Q1]` — 1 câu hỏi cụ thể
- `[ANSWER 1]` — 1-2 câu trả lời có data/ví dụ
- Transition: "Nhưng..." / "Vấn đề là..." / "Tới đây bạn sẽ hỏi..." → mở Micro-Q 2

### Step 5 — Micro-Q 2 block (15-30s)

Cùng cấu trúc. Answer 2 phải reveal 1 nuance mà Answer 1 chưa chạm. Transition sang Q3.

### Step 6 — Micro-Q 3 block (30-45s)

Layer sâu nhất, thường là counter-argument hoặc edge case. Answer 3 set up cho payoff.

### Step 7 — Final payoff (45-55s)

Viết câu này TRƯỚC TIÊN (Last Dab rule). Payoff phải:
- Tie 3 layer lại thành 1 verdict cho MAIN Q
- Không hype, không values-based ("tốt nhất", "phải")
- 1-2 câu, sticky

### Step 8 — Soft CTA (55-60s)

Benevolent hoặc Laissez-faire:
- "Mình thử tiếp phần 2, follow để xem."
- "AI Freedom Builders có thread chi tiết — link bio."

### Step 9 — Visual + audio direction

Cho mỗi section, note:
- Camera: talking head / screen recording demo / b-roll
- Caption: bold stroke, highlight keyword của MICRO Q
- Music: drop khi hỏi, rising khi answer, silence khi payoff
- Pacing: mỗi layer = 1 cut, tension rising

## Mandatory Rules

- [ ] Duration 30-60s (default 60s)
- [ ] Hook ≤3s, mở bằng MAIN Q (không mở bằng answer)
- [ ] Đúng 3 layer Micro-Q (không ít hơn, không nhiều hơn)
- [ ] Mỗi answer có 1 data point specific hoặc ví dụ cụ thể
- [ ] Transition giữa layer phải "grow" từ answer trước (không rời rạc)
- [ ] Payoff tie hết 3 layer về MAIN Q
- [ ] Last Dab viết trước — hook và payoff align
- [ ] CTA non-autocratic (không "Follow ngay!")
- [ ] Brand voice 7/10 — không hype ("INSANE", "crazy")
- [ ] Power words English giữ: System, Automation, AI, Framework, Workflow, Prompt, Template
- [ ] Nhân xưng "mình/bạn", không "tôi/các bạn"
- [ ] Script marker có đủ: `[MAIN Q]`, `[MICRO Q1-3]`, `[ANSWER 1-3]`, `[FINAL PAYOFF]`
