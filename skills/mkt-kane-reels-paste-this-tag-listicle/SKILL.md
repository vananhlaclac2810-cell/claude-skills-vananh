# Reels — Paste-this-tag Listicle Format

Format **validated 2+ lần vào Gold** trên kênh Sabrina Ramonov (2M+ followers). Skeleton G1 + G3 khác 100% nội dung nhưng cùng cấu trúc — confirm đây là format thật, không phải lucky hit.

**Core insight**: Viewer coi Reel như "collectible cheat sheet" — paste ngay được, save để dùng sau. 4 lớp tension đồng thời: (1) số cụ thể promise N cái, (2) lead magnet ép comment, (3) UPPERCASE tag tạo pattern visual dễ nhớ, (4) "Save this" cho lý do hợp lý để bookmark.

## Khi nào dùng format này

✅ **Dùng khi**:
- Topic có tập hợp N phần tử paste-able (5 prompts, 5 Claude Code hooks, 5 shortcut, 5 settings)
- Mỗi phần tử có thể tóm trong 1 dòng benefit
- Có lead magnet ready (template file, prompt library, guide PDF)
- Pillar P1 (AI Demo) hoặc P2 (OPB workflow)

❌ **KHÔNG dùng khi**:
- Topic kể chuyện / mindset / behind-the-scenes → dùng `mkt-kane-reels-untold-stories` hoặc `mkt-kane-reels-jenga-tension`
- Chỉ có 1-2 phần tử (không đủ listicle) → dùng `mkt-kane-reels-visual-metaphor`
- Không có lead magnet → giảm 60% upward driver (H2)

## Structure (7 beats, 45-60s)

| Beat | Thời lượng | Chức năng |
|------|-----------|-----------|
| **1. Lead magnet CTA** | Caption line 1 (viewer đọc trước khi play) | Force comment → engagement signal |
| **2. Number hook** | 0-3s | Promise số cụ thể + outcome đo được |
| **3. Setup line** | 3-6s | "Paste vào [context]:" — chuyển sang listing mode |
| **4. N tag blocks** | 6-45s | Mỗi block 5-8s: UPPERCASE TAG → 1-line benefit → micro example/demo |
| **5. Loop tease** | 45-50s | "Còn [N+5] cái nữa — comment [KEYWORD] mình gửi hết" |
| **6. Collectible close** | 50-55s | "Save bài này để lần sau còn paste" |
| **7. Soft CTA** | 55-60s | Non-autocratic follow/community invite |

## Input

User cung cấp:
1. **AI tool / context** — vd: "Claude Code", "ChatGPT", "Cursor", "Gemini"
2. **Theme** (loại tag) — vd: "hooks", "prompts", "settings", "commands", "slash commands"
3. **Số N** (default 5, chấp nhận 3-7)
4. **Lead magnet resource** — vd: "template settings.json đầy đủ", "prompt library", "guide setup"
5. **Keyword lead magnet** — 1 từ, dễ type, viết hoa, 4-7 ký tự (vd: `HOOKS`, `CLAUDE`, `SETUP`, `PROMPT`)

Nếu user thiếu input, hỏi nhanh 1 lần. Không hỏi hơn 3 câu.

## Process (Steps)

### Step 1 — Viết Last Dab TRƯỚC TIÊN

Theo brand voice rule. Last Dab cho format này thường là 1 trong 3 pattern:
- **Ownership flex**: "Mình dùng từ tháng [X] — giờ không viết prompt thô nữa."
- **Contrarian punchline**: "Ai còn type prompt thô là đang lãng phí [AI tool] rồi."
- **Before/after sticky**: "Prompt dài 200 chữ giờ còn 20 — vẫn ra output y hệt."

Last Dab này sẽ quyết định tone của cả script. Viết 3 variant cho user chọn.

### Step 2 — Xác định N tag + benefit 1-line

Cho mỗi tag, cần:
- **Tag name** — UPPERCASE nếu English (TRUTHMODE, PARETO), **snake_case hoặc PascalCase** nếu là tên technical thực (PostToolUse, PreToolUse) — **KHÔNG dịch tag name sang tiếng Việt**
- **1-line benefit** — bắt đầu bằng **động từ mạnh** tiếng Việt ngắn: "Cắt 70% token", "Block mọi lệnh xóa", "Auto format code sau edit", "Warn trước khi commit .env"
- **Micro example** (optional, 3-5 từ) — để demo trên screen: `rm -rf / → blocked`, `prompt 50 chars → 500 words output`

**Rule**: benefit phải đo được (số, action, outcome) — không abstract ("giúp năng suất hơn" ❌).

### Step 3 — Viết Number hook (0-3s)

Pattern bắt buộc: `[SỐ] [ĐƠN VỊ] [ĐỘNG TỪ MẠNH] [AI TOOL] [X] [OUTCOME ĐO ĐƯỢC]`

Đúng cách:
- ✅ "5 Claude Code hooks cắt 3 tiếng debug mỗi ngày"
- ✅ "4 prompts làm ChatGPT ra output x10 chất lượng"
- ✅ "3 slash command mình type 50 lần/ngày trong Claude Code"

Sai cách (vague, values-based):
- ❌ "Claude Code hooks cực mạnh bạn phải thử" (thiếu số + thiếu outcome đo được)
- ❌ "5 tips làm AI tốt hơn" (vague "tốt hơn")
- ❌ "Những Claude Code hook INSANE" (hype word)

Viết 3 variant hook cho user chọn — cùng cấu trúc, khác từ/khác outcome đo.

### Step 4 — Viết Lead magnet CTA (caption line 1)

Pattern: `Comment '[KEYWORD]' [và/để] mình gửi [resource đầy đủ] 👇`

Ví dụ:
- `Comment 'HOOKS' để mình gửi file settings.json full 12 hooks 👇`
- `Comment 'CLAUDE' mình share prompt library 50+ template tiếng Việt 👇`

**Rules**:
- KEYWORD ≤ 7 ký tự, dễ type trên mobile VN
- Emoji 👇 (down arrow) ở cuối — match Sabrina pattern
- `mình` không phải `tôi`/`admin`
- Resource phải cụ thể, không "full course" / "everything you need"

### Step 5 — Setup line (3-6s)

Transition từ hook vào listing. Pattern:
- "Paste vào [file/context]:" — vd: "Paste vào `~/.claude/settings.json`:"
- "Dùng trước mỗi prompt:" — khi tag là prompt prefix
- "Bật lên là xong:" — khi tag là toggle/flag

Câu này nói rõ WHERE paste + HOW dùng. Viewer cần biết paste ở đâu để lý do save mới đủ.

### Step 6 — N tag blocks (6-45s)

Mỗi block cấu trúc:
```
[TAG_NAME_UPPERCASE]
[1 dòng benefit - bắt đầu bằng động từ]
(optional: [micro example 3-5 từ])
```

Ví dụ cho Claude Code hooks:
```
PostToolUse
Auto format code mỗi lần Claude edit.
→ prettier chạy ngầm, không cần nhớ.

PreToolUse
Block lệnh xóa file khi chưa review.
→ rm -rf /tmp/* → blocked.
```

**Pacing rule**: Mỗi tag 5-8s. Visual: tên tag BIG CENTERED, benefit animate-in dưới.

### Step 7 — Loop tease (45-50s)

Tạo lý do comment cho người đã xem full:

Pattern:
- "Còn [N+3] hooks nữa — comment '[KEYWORD]' mình gửi hết."
- "Đây mới là 5/15. File đầy đủ mình tặng ai comment '[KEYWORD]'."

Phải match lead magnet ở caption line 1. KHÔNG promise khác với caption.

### Step 8 — Collectible close (50-55s)

Pattern:
- "Save bài này để lần sau setup còn paste."
- "Save đi — mấy hook này 1 tháng mới cần đụng 1 lần nhưng cứu mạng."

**Insight**: "Save this" cho viewer LÝ DO hợp lý để bookmark → tăng save rate → FB algorithm đẩy mạnh.

### Step 9 — Soft CTA (55-60s)

Non-autocratic. Pick 1 trong 3:
- **Benevolent**: "Follow để coi tuần sau mình ra bộ 5 hook nữa cho automation."
- **Laissez-faire**: "Ai xài Claude Code mà chưa set hook — tự xem thử có thiếu không."
- **Community invite**: "AI Freedom Builders có thread đủ 15 hook + setup — link bio."

KHÔNG dùng: "Follow ngay!", "Subscribe liền!", "Phải share!"

### Step 10 — Visual + Audio direction

Cho mỗi section, note:
- **Camera**: screen recording preferred (paste-able content cần show code/UI)
- **Overlay text**: tag UPPERCASE big font, benefit bold stroke (TikTok-style caption)
- **Music**: sub-bass low-energy lo-fi, volume drop khi đọc tag → viewer focus
- **Pacing cut**: 1 cut per tag, mỗi cut 5-8s
- **Transition**: swipe/slide, KHÔNG dùng animation fancy (anti-downward driver #2 over-production)
