# Reels — Is it Worth It? Format (Tanner Leatherstein style)

Brendan Kane case study: Tanner Leatherstein (leather craftsman, 1.7M followers TikTok/IG). Signature: mua túi luxury (LV, Hermes, Gucci) → cắt đôi → phân tích leather quality + stitching + hardware → verdict giá thực vs giá bán.

**Core insight**: Viewer nghi ngờ giá luxury nhưng không có data. Video cho họ forensic analysis + verdict fair → share vì "finally someone gave me the truth". KHÔNG phải hate-review — fair breakdown.

## Structure (6 sections, ~45-90s total)

1. **Hook** (0-3s) — "Bạn có đang trả quá nhiều cho [X]?"
2. **Deconstruct** (3-30s) — break down component/spec/module
3. **Market comparison** (30-55s) — giá bán vs giá trị thật (specific numbers)
4. **Verdict** (55-75s) — worth it / not worth it with reasoning
5. **Alternative recommendation** (75-85s) — đề xuất thay thế fair
6. **Soft CTA** (85-90s) — non-autocratic

## Input

1. **Product / tool / course name** (vd: "Khóa AI Agent $500", "ChatGPT Plus $20/tháng", "Cursor IDE $20/tháng")
2. **Giá bán** (exact number + currency + period)
3. **Public features/specs** (list những gì product claim)
4. **Duration** (default 75s)

## Process

### Step 1 — Commit to FAIRNESS

Trước khi viết: quyết định verdict. NHƯNG verdict phải đến từ data, không bias. Rule:
- KHÔNG hate-review (không "scam", "lừa đảo")
- KHÔNG sponsored tone (không "phải mua ngay")
- KHÔNG vague ("đắt", "rẻ") — phải specific number

Nếu bạn chưa dùng product → mark rõ "based on public info" + recommend user test.

### Step 2 — Write Last Dab FIRST

Verdict 1 câu ≤15 từ. Tie specific number.

VD:
- "Khóa $500 này giá trị thật $200. Worth it nếu bạn là beginner. Skip nếu bạn biết rồi."
- "ChatGPT Plus $20 đáng — nhưng chỉ nếu bạn dùng 40+ prompt/ngày."
- "Cursor $20 = VS Code + Copilot $10 + 10% productivity. Worth it cho full-time dev."

### Step 3 — Hook 0-3s

Hook pattern (chọn 1):
- "Bạn có đang trả quá nhiều cho [X]?"
- "[X] giá $[N]. Nó đáng không? Mình mổ xẻ 60 giây."
- "Mình đã thử [X] 30 ngày. Đây là data thật."
- "Trước khi bạn mua [X], xem video này."

Viết 3 hook variations cho user chọn.

### Step 4 — Deconstruct (3-30s)

Break product thành component. Mỗi component:
- Spec/feature cụ thể
- Giá trị estimate (số cụ thể)

Format cho course/software:
- **Module / feature breakdown**: "Khóa có 12 module × 2h = 24h content. Instructor có X năm kinh nghiệm. 3 template kèm."
- **Per-unit cost**: "24h content ÷ $500 = $20/h. Udemy trung bình $5/h."

Format cho physical/subscription:
- **Component breakdown**: "ChatGPT Plus = GPT-5 access + DALL-E + Advanced Voice + 80 msg/3h."
- **Per-feature cost**: "Nếu dùng 40 msg/ngày = 1200 msg/tháng. $20 ÷ 1200 = $0.016/msg."

Visual: screen recording, pricing page, feature list, ASMR nếu product physical.

### Step 5 — Market comparison (30-55s)

Compare với 2-3 alternatives. Rules:
- Cùng category
- Cùng tier (không so luxury vs budget)
- Specific number diff (không "cheaper" — phải "$X cheaper" hoặc "X% cheaper")

VD:
> "So với Claude Pro: cùng $20, Claude cho 5x limit + 200K context. ChatGPT cho DALL-E + Voice. Claude win writing + coding. ChatGPT win multimodal."

### Step 6 — Verdict (55-75s)

Verdict format:
- **Worth it IF**: specific user type + use case
- **NOT worth it IF**: specific user type + use case

VD:
> "Worth it nếu bạn: (1) viết content 2h+/ngày, (2) chưa quen Claude interface, (3) cần Voice + DALL-E trong workflow.
> NOT worth it nếu: (1) bạn dev full-time — Claude Pro tốt hơn, (2) bạn dùng <5 prompt/ngày — free tier đủ."

### Step 7 — Alternative recommendation (75-85s)

Đề xuất 1 alternative fair. Không push product của mình aggressive.

VD:
> "Thay vào đó: Claude Pro $20 nếu dev, hoặc combo ChatGPT Free + Claude Free cho beginner."

Nếu có product của Hoang (AI Freedom Builders, khóa) — chỉ mention ngang, không hype.

### Step 8 — Soft CTA (85-90s)

Benevolent hoặc Laissez-faire:
- "Bạn đã thử [X]? Comment verdict của bạn."
- "Muốn review tool nào tiếp? Drop name dưới."
- "AI Freedom Builders có bảng so sánh AI tool — link bio."

### Step 9 — Production notes

- Camera: pricing page screen recording + face reaction
- Nếu product physical: ASMR angle (close-up, sound raw)
- Pacing: nhanh phần deconstruct, chậm phần verdict
- On-screen text: giá + specific number (viewer screenshot được)
